
# 我们的项目：农业温室大棚"环境监测+自动控制"闭环系统

## 用户故事视角

| 用户故事 | 角色 | 功能模块 | 功能名 | 描述 | 功能流程 |
|------|------|------|------|------|------|
| 作为用户，我希望能实时查看温湿度数据，以便了解植物当前生长环境 | 用户 | 数据监测 | 温湿度监测 | 实时展示当前温度和湿度数值 | 温湿度传感器采集 → MQTT 上报 → 后端接收处理 → 前端页面展示 |
| 作为用户，我希望能实时查看光照强度，以便判断植物是否需要补光 | 用户 | 数据监测 | 光照监测 | 实时展示当前光照强度及光照状态 | 光照传感器采集 → MQTT 上报 → 后端接收处理 → 前端页面展示 |
| 作为用户，我希望能查看通风和补光状态，以便了解系统是否正在自动调节环境 | 用户 | 数据监测 | 设备运行状态监测 | 展示风扇是否开启、补光灯是否开启及当前运行状态 | 设备状态采集/回传 → MQTT 上报 → 后端解析 → 前端状态展示 |
| 作为用户，我希望能查看历史环境变化趋势，以便分析植物生长情况 | 用户 | 数据可视化 | 历史数据趋势 | 以图表方式展示温度、湿度、光照等历史变化趋势 | 后端查询历史监测数据 → 前端图表组件渲染 |
| 作为用户，我希望能在日历中查看每天的植物图片和记录，以便回顾历史状态 | 用户 | 数据可视化 | 日历日志查看 | 用日历方式展示每日植物图片、环境数据和事件记录 | 后端查询每日图片与日志 → 前端日历渲染 → 点击日期查看详情 |
| 作为用户，我希望能远程控制补光灯，以便在光照不足时手动补光 | 用户 | 设备控制 | 补光灯控制 | 通过网页远程开启或关闭补光灯 | 前端发送控制指令 → 后端转发 MQTT 指令 → 硬件执行 → 状态回传前端 |
| 作为用户，我希望能远程控制风扇，以便在环境闷热时手动通风 | 用户 | 设备控制 | 风扇控制 | 通过网页远程开启或关闭风扇 | 前端发送控制指令 → 后端转发 MQTT 指令 → 硬件执行 → 状态回传前端 |
| 作为用户，我希望能设置环境阈值，以便系统在不合适时自动调节和提醒 | 用户 | 策略管理 | 阈值告警策略 | 设置温湿度、光照等阈值，超限时自动告警或触发控制 | 用户设置阈值 → 后端保存规则 → 实时数据进入判断 → 触发控制或告警 |
| 作为用户，我希望能设置自动补光和自动通风规则，以便实现基础自动化照护 | 用户 | 策略管理 | 自动控制策略 | 根据环境状态自动控制补光灯和风扇 | 用户配置自动规则 → 后端保存策略 → 实时数据触发判断 → 自动下发控制指令 |
| 作为用户，我希望能查看策略触发记录，以便追溯系统自动控制情况 | 用户 | 数据可视化 | 策略执行日志 | 查看自动补光、自动通风、异常告警等策略执行历史 | 后端查询策略日志 → 前端列表展示 |
| 作为用户，我希望能收到植物异常提醒，以便及时处理倾斜、空气异常等问题 | 用户 | 告警管理 | 异常告警 | 当植物倾斜、空气异常或烟雾异常时提醒用户 | 倾斜/烟雾/气体模块采集 → MQTT 上报 → 后端判断异常 → 前端弹出告警并记录日志 |
| 作为用户，我希望能查看异常历史记录，以便追溯植物生长过程中的问题 | 用户 | 告警管理 | 告警日志查看 | 查看历史异常、危险状态和处理记录 | 异常事件写入日志 → 后端存储 → 前端列表展示 |
| 作为用户，我希望能知道今天有没有人来看过植物，以便增加互动感和陪伴感 | 用户 | 陪伴互动 | 查看事件记录 | 展示主人是否靠近过植物，以及查看次数 | 人体红外模块检测靠近 → 上报事件 → 后端记录 → 前端显示“主人来了”及查看次数 |
| 作为用户，我希望能查看植物每日照片日志，以便观察植物每天的成长变化 | 用户 | 图片日志 | 每日植物图片日志 | 上传并展示每日植物图片，形成成长记录 | 用户上传照片 → 后端调用 SmartAI 进行植物主体识别与分割 → 生成统一展示图片 → 保存日志 → 前端按日历展示 |
| 作为用户，我希望能在日历中查看/设置每天的植物图片和记录，以便回顾历史状态 | 用户 | 数据可视化 | 日历日志查看 | 用日历方式展示每日植物图片、环境数据和事件记录 | 后端查询经 SmartAI 植物主体识别处理后的图片与日志 → 前端日历渲染 → 点击日期查看详情 |
| 作为用户，我希望能通过对话咨询植物养护问题，以便获得照护建议 | 用户 | 智能问答 | 植物知识问答 | 通过 RAG 问答获取植物养护、病害识别、环境调节建议 | 用户提问 → 知识库检索 → 大模型生成回答 → 前端展示答案 |
| 作为用户，我希望能设置定时开关补光灯，以便定时补光 | 用户 | 策略管理 | 定时控制策略 | 设置补光灯的定时开关时间 | 后台设置定时规则→定时触发→下发指令→硬件执行 |
| 作为管理员，我希望能查看系统告警记录，以便追溯系统运行中的问题 | 管理员 | 告警管理 | 系统告警日志 | 查看所有告警事件、告警类型及处理状态 | 后端查询告警日志 → 管理端列表展示 |
